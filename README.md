# Codename: KV2

<sub>The name will change when I think of a better one.</sub>

This project is a web application to help track your roster in a wargaming campaign.

## Status

This is pre-release software under active development directly against `main`. There is no 'production' deployment yet.

## Documentation

Design documents can be found [in the docs folder](https://github.com/edward3h/kv2/tree/main/docs).

## Architecture

* Server: [Avaje Inject](https://avaje.io/inject/) (dependency injection) + [Avaje Jex](https://avaje.io/jex/) (HTTP), running on Java.
* Database: MySQL, accessed via DAOs generated by [kiwiproc](https://github.com/edward3h/kiwiproc), with schema migrations managed by Liquibase.
* Views: server-rendered [JTE](https://jte.gg/) templates, styled with Tailwind CSS and DaisyUI (`views` subproject).
* Browser client: `teaClient`, compiled from Java to JavaScript with [TeaVM](https://teavm.org/), for browser-side interactivity.
* `rosterParser`: shared Java parsing library used by both the server and `teaClient`.
* Auth: JWT-based sessions with OAuth2 sign-in (currently Discord); a dev-only stub OAuth provider lets you run and test locally without real provider credentials.

See [docs/architecture.md](docs/architecture.md) for more detail.

## Getting Started

### Requirements

* Java, Node.js and [pnpm](https://pnpm.io/installation), at the versions pinned in [`.tool-versions`](.tool-versions). [asdf](https://asdf-vm.com/) is the recommended tool for managing these.
* Docker, to run MySQL locally via `docker-compose.yml`.

### Setup

Create a `.env` file in the project root with at least:

* `JWT_SECRET` - any non-empty value for local development.
* `DISCORD_CLIENT_ID` / `DISCORD_CLIENT_SECRET` and `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` - only required if you want to test against the real OAuth providers. For everyday development, `./gradlew run` automatically points OAuth at an embedded stub provider, so these can be left blank.

### Running

```
docker compose up -d --wait mysql   # or: ./gradlew startDevMySQL
./gradlew run
```

The app starts on http://localhost:8080.

### Testing

* `./gradlew test` - unit and integration tests (uses Testcontainers for MySQL).
* `./gradlew check` - the above, plus the Playwright-based UI test suite.
